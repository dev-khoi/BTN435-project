Network Design & Simulation • ☐ Network topology diagram is complete and clearly labeled • ☐ Subnetting or VLSM is correctly designed and applied • ☐ IP addressing scheme is logical (organized in meaningful way) • ☐ Multiple subnets are implemented (e.g., lighting, security, thermostat) • ☐ Devices can communicate across subnets successfully
2. Client–Server Application (C++) • ☐ Server program is fully implemented and running • ☐ Server handles multiple clients simultaneously • ☐ Client application is functional • ☐ Client can send commands to the server • ☐ Server processes requests and sends correct responses • ☐ TCP socket communication is stable and reliable • ☐ Routing (static/default) functionality (simulation) is functional • ☐ ARP functionality (simulation) is implemented
3. Application Protocol • ☐ Custom HTTP-like protocol is implemented • ☐ Request format is clearly defined (e.g., GET /device/action) • ☐ Response format includes status/confirmation/error messages • ☐ Commands are correctly parsed and executed • ☐ Invalid commands are handled properly
4. Device Simulation • ☐ At least 3 types of smart devices are implemented (e.g., lights, thermostat, security camera) • ☐ Devices support ON/OFF functionality and respond correctly to commands from the server • ☐ Devices allow status checking from the server and client(s) • ☐ Devices support configuration options (e.g., temperature setting, brightness level) • ☐ Device states are maintained correctly across multiple client terminals • ☐ Multiple client instances can connect simultaneously and control devices without conflicts • ☐ Device responses are synchronized and reflect real-time changes across all clients
5. Multithreading & Concurrency • ☐ Multithreading is implemented in the server • ☐ Multiple clients are handled concurrently • ☐ Device operations can run simultaneously • ☐ Shared resources are synchronized (mutex/locks used) • ☐ No race conditions, deadlocks, or crashes observed
6. Routing & Network Communication • ☐ Communication works across different subnets • ☐ Routing logic is correctly implemented • ☐ ARP resolves IP to MAC for local communication • ☐ (If implemented) ICMP or advanced routing works correctly
7. System Integration & Testing • ☐ Entire system works end-to-end • ☐ Multiple clients + multiple devices tested • ☐ Edge cases tested (invalid input, disconnects, etc.) • ☐ Errors are handled gracefully • ☐ System runs reliably without failure
8. Code Quality • ☐ Code is clean, modular, and well-organized • ☐ Meaningful variable/function names used • ☐ Proper comments included • ☐ No compilation/runtime errors • ☐ Project structure is easy to navigate

Project Report (15–20 Pages) • ☐ Introduction and objectives included • ☐ Network design explained (topology, subnetting, routing) • ☐ System architecture described (client-server) • ☐ Protocol design explained • ☐ Key code sections explained • ☐ Testing methodology and results included • ☐ Challenges and solutions discussed • ☐ Diagrams and screenshots included • ☐ Proper formatting and organization
